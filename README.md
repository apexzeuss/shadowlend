# ShadowLend

Cross-asset lending on Solana devnet. Real Anchor program, real SPL transfers,
real Pyth-priced cross-asset health checks. The privacy layer (Arcium MXE) is
on the roadmap, not yet wired.

> Program ID: `5jqXbgExBEnKPahsQineFmMJHNcEvwnniiYvDy81bZCF`  ·  Cluster: devnet
>
> Live site: pushes from `main` auto-deploy.

---

## What works today

- **Three markets** with their own test SPL mints, vaults, and Pyth feeds:
  SOL (80% max LTV), USDC (85%), BTC (75%).
- **One-claim-per-wallet faucet** per market. Each market issues its own test
  token, 9 decimals.
- **Supply / borrow / withdraw / repay** as real on-chain SPL transfers into
  per-market PDA-owned vaults.
- **Cross-asset health check** at every borrow and withdraw. The program walks
  every Position the user has, prices each via Pyth's `PriceUpdateV2`, sums
  collateral × per-market LTV against total debt.
- **Same-mint borrow disabled.** You can supply SOL, but you can't borrow SOL
  against it. Forces real cross-asset collateralisation.
- **Eager-reconnect** in the frontend. Wallet survives page reloads via
  `localStorage` + `adapter.connect({ onlyIfTrusted: true })`.
- **Real activity stream.** The proof list and stat tiles read fresh from
  `getSignaturesForAddress` and on-chain Market accounts, no fabricated
  numbers.

## What is not real yet

- **Privacy layer.** The Arcium MXE flow is described in the "Privacy
  preview" section as planned, not live. Every supply and borrow is a public
  SPL transfer on devnet right now.
- **Liquidation.** A position with health < 1 stays undercollateralised
  without recourse. Coming next.
- **Interest accrual.** Vaults don't earn yield, debt doesn't grow. Coming
  after liquidation.
- **Upgrade authority.** The program is upgradeable by the single admin
  keypair. Fine for devnet, must move to a multisig before mainnet.

## Build

### Prerequisites

- Anchor CLI `0.30.1`
- Solana CLI `2.1.x` (Agave client)
- Node 18+ and Yarn

### One-time setup

```sh
yarn install
solana config set --url devnet
solana airdrop 2          # repeat until you have ~5 SOL
```

### Build the program

```sh
anchor build              # produces target/deploy/shadow_lend.so
node scripts/build-idl.js # rebuilds the hand-written IDL
```

### Deploy + initialise markets (one-time per cluster)

```sh
solana program deploy \
  target/deploy/shadow_lend.so \
  --program-id target/deploy/shadow_lend-keypair.json \
  --url devnet

TS_NODE_TRANSPILE_ONLY=true \
ANCHOR_PROVIDER_URL=https://api.devnet.solana.com \
ANCHOR_WALLET=~/.config/solana/id.json \
  yarn ts-node migrations/initialize-markets.ts
```

The migration writes `app/config.js` with the resulting mint addresses.

### Verify on devnet

```sh
TS_NODE_TRANSPILE_ONLY=true \
ANCHOR_PROVIDER_URL=https://api.devnet.solana.com \
ANCHOR_WALLET=~/.config/solana/id.json \
  yarn ts-node migrations/smoke-test.ts
```

### Run the frontend locally

```sh
cd app && python3 -m http.server 8765
```

Open `http://localhost:8765` and connect a wallet on Solana devnet.

## Build gotchas (the ones that will trip you)

These are landmines this codebase has already worked around. If you see one,
this is the answer.

### `anchor-syn` fails with "no method named `source_file`"

Anchor's IDL extraction calls `proc_macro2::Span::call_site().source_file()`,
which is a nightly-only API gated behind `procmacro2_semver_exempt`. On the
stable toolchain this errors out and `anchor build` reports "Building IDL
failed."

**The `.so` still builds.** Only the IDL extraction step fails. We hand-maintain
the IDL in `scripts/build-idl.js`. Run it after every program change. The JSON
it writes is byte-compatible with what Anchor would emit, because all the
discriminators are computed the same way (sha256 of `account:Name`,
`global:ix_name`, etc).

### `edition2024` cascade in transitive deps

Solana platform-tools v1.43 ships `rustc/cargo 1.79`, which can't parse crates
that require `edition2024`. `Cargo.lock` pins a few deps to pre-edition2024
versions:

- `proc-macro-crate@3.2.0`
- `blake3@1.5.5` (newer `cpufeatures` needs edition2024)
- `indexmap@2.7.0`
- `unicode-segmentation@1.12.0`

If a fresh `cargo update` brings the issue back, downgrade with
`cargo update -p NAME --precise OLDVERSION`. A more durable fix is upgrading
platform-tools (newer Solana CLI ships newer cargo).

### Stack frame overflow on Anchor `init` contexts

Anchor 0.30 generates `try_accounts` functions whose frame size includes every
account validation inline. With more than three or four `init`/`init_if_needed`
accounts plus `Sysvar<Rent>`, the BPF stack frame (4096 bytes) overflows.

Mitigations used here:

1. `Box<Account<'info, T>>` for every non-trivial account.
2. Dropped `rent: Sysvar<'info, Rent>` — Anchor 0.30 uses `Rent::get()`
   internally for `init`, so the explicit account is redundant.
3. Merged per-mint `mint_authority` and `vault_authority` PDAs into one
   `authority` PDA per market.

If you add an account to an `init` context and the build prints
"Stack offset of N exceeded max offset of 4096", apply (1) to the new account
first.

### Devnet airdrop rate limits

`solana airdrop 2 --url devnet` will rate-limit you fast. If you need a buffer
of SOL for a redeploy (~2.5 SOL goes into the buffer rent during an upgrade),
use [faucet.solana.com](https://faucet.solana.com) instead it has a separate
rate limit and accepts captcha.

### Pyth `PriceUpdateV2` addresses rotate on devnet

The sponsored shard-0 PDA derivation that works on mainnet does not match the
posted accounts on devnet. We resolve at runtime via
`getProgramAccounts(pyth_receiver, [dataSize:134, memcmp@41=feed_id])` and
pick the most recent `publish_time`. See `resolveFreshestPrice` in
`app/sl-client.js`.

## Architecture

### Program accounts

```
Market[mint]            ← per-market config (admin, mint, faucet amount, LTV,
                          feed_id, totals, bumps). Seeds [b"market", mint].
Position[user,mint]     ← per-(wallet, market) state (supplied, borrowed,
                          denorm. LTV+decimals+feed_id, bump).
                          Seeds [b"position", user, mint].
ClaimReceipt[user,mint] ← one-shot proof of faucet claim.
                          Seeds [b"claim", user, mint].
UserStats[user]         ← aggregate action counters.
                          Seeds [b"stats", user].
Authority[mint]         ← PDA that signs as mint authority and vault owner.
                          Seeds [b"auth", mint].
Vault[mint]             ← token account owned by Authority[mint].
                          Seeds [b"vault", mint].
```

### Instruction surface

| Instruction        | Who   | Effect                                            |
|--------------------|-------|---------------------------------------------------|
| `initialize_market`| admin | Creates Market, mint, vault, sets feed_id + LTV   |
| `set_claim_amount` | admin | Adjusts faucet payout per claim                   |
| `claim_faucet`     | user  | Mints `amount_per_claim` to caller's ATA          |
| `supply`           | user  | Tokens from user → vault, position.supplied +=    |
| `withdraw`         | user  | Vault → user, with cross-asset health check       |
| `borrow`           | user  | Vault → user, with cross-asset LTV check          |
| `repay`            | user  | User → vault, position.borrowed -= amount         |
| `init_user_stats`  | user  | One-time init of UserStats                        |
| `record_action`    | user  | Bumps action counters in UserStats                |

`borrow` and `withdraw` take `remaining_accounts` = pairs of `(position,
price_update)` for every *other* market the user has activity in. The program
verifies each position is owned by `crate::ID`, that the user matches, and that
the price_update's feed_id matches the position's denormalised feed_id.

### Frontend boot

```
config.js loads first (defines window.SHADOW_LEND_CONFIG)
sl-client.js (ES module) loads, dispatches 'sl-client-ready'
index.html boot:
  loadMarketData()          ← fetches Market PDAs for the cards
  refreshHeroStats()        ← TVL, borrowed, prices via Pyth
  refreshActivityStats()    ← open positions count, totals
  loadRecentProofs()        ← getSignaturesForAddress + per-tx decode
  tryEagerReconnect()       ← localStorage + onlyIfTrusted connect
```

After connect, `refreshUserState()` reads every Position for the wallet,
prices them, and computes one global health factor.

## Roadmap

Tracked as work items, not promises:

- **Arcium MXE on the LTV path.** Real circuit, real cluster, real MPC verify.
- **Liquidation.** Anyone can close an undercollateralised position for a
  discount on the seized collateral.
- **Interest accrual.** Per-slot utilisation-driven rate, compounded into
  Position state.
- **Aggregate ProtocolStats PDA.** Replace the open-positions
  `getProgramAccounts` call with one Account read.
- **Multisig upgrade authority.** Single-keypair upgrades are devnet-only.
- **Split `app/index.html`.** Single 3.4k-line file is fine for now, not for
  the next contributor.
