04 / 07 — PROOF FLOW
04 · How a borrow clears

Four steps. Zero balance leaks.

Your number flows through encryption, secure compute, a yes/no proof, and on-chain settlement — and at every step, only an encrypted value or a boolean is visible.

1
Encrypt locally

Your amount is encrypted in the browser before any request fires. The plain number never crosses the wire.

browser
2
MXE computes

Arcium's private compute layer checks the rule LTV bound, health factor, supply cap on the encrypted amount.

arcium
3
Return a proof

Only the boolean result is published. The program sees whether the action is valid, never your underlying balance.

proof
4
Settle on Solana

The position is updated using encrypted state and proof refs. Public state never reveals the size of your position.

solana
